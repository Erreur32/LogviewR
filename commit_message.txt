release: v0.8.34 — security hardening

## For users
- JWT pinned to HS256 — prevents alg:none bypass attacks
- Logout endpoint revokes token server-side (POST /api/users/logout)
- HSTS + CSP security headers on all responses
- Login rate-limited (10 req/min) as defense-in-depth
- Webhook URLs validated against private/internal IPs (SSRF protection)

## Technical
- authService: jwt.verify with { algorithms: ['HS256'] }, token blacklist with 10min cleanup
- users.ts: POST /logout route, rateLimit on POST /login
- index.ts: Strict-Transport-Security (prod), Content-Security-Policy (Leaflet tiles, GitHub API, WSS)
- system.ts: /security-status now requires auth
- log-viewer.ts: null byte check in validatePathSafe, ReDoS protection for user regex
- notifications.ts: validateWebhookUrl blocks localhost, RFC1918, cloud metadata
- systemServer.ts: quoted HOST_ROOT_PATH in chroot commands
- Vite 6.2.0 → 6.4.2 (2 CVE fixes)
- .gitignore: added CLAUDE.md, resume_claude*.txt, data-test/, .planning/
