# IPTables/IPSet/NFTables Docker Fix — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Faire fonctionner les onglets IPTables, IPSet et NFTables dans le container Docker (identique à `npm run dev`).

**Architecture:** Deux prérequis cumulatifs sont nécessaires :
1. `network_mode: host` — le container partage le namespace réseau du host → les règles fail2ban sont visibles.
2. `cap_add: NET_ADMIN` — capacité Linux requise par le kernel pour lire/écrire netfilter (iptables/ipset/nft), même en root dans un container.
3. `sudo` dans `Fail2banClientExec.runBin()` — l'app tourne en `node` user (via su-exec), il faut élever les droits. Le sudoers est déjà configuré dans le Dockerfile.

**Tech Stack:** Docker Compose, Alpine Linux, iptables-nft, ipset, nftables, Node.js (su-exec node)

**Diagnostic effectué :**
- `network_mode: host` SEUL → `Permission denied` même en root (CAP_NET_ADMIN manquant)
- `network_mode: host` + `cap_add: NET_ADMIN` → `iptables-save` et `ipset list` retournent les règles host ✓
- `Fail2banClientExec.runBin()` appelle `ipset`/`nft` sans sudo → échoue en `node` user ❌
- `IptablesService` a déjà `priv()` (sudo) → TabIPTables probablement déjà OK ✓

### Task 2 : Ajouter `sudo` dans `Fail2banClientExec.runBin()`

**Files:**
- Modify: `server/plugins/fail2ban/Fail2banClientExec.ts`

`runBin()` appelle directement les binaires réseau (`iptables-save`, `ipset`, `nft`) sans sudo.
L'app tourne en `node` user → échec. Le sudoers dans le Dockerfile couvre déjà ces binaires.

- [ ] **Step 1 : Ajouter la fonction `priv()` et corriger `runBin()`**

En haut du fichier (après les imports, avant la classe), ajouter :

```typescript
/** Prepend sudo when not running as root (same pattern as IptablesService). */
function priv(cmd: string, args: string[]): [string, string[]] {
    const isRoot = typeof process.getuid === 'function' && process.getuid() === 0;
    return isRoot ? [cmd, args] : ['sudo', ['-n', cmd, ...args]];
}
```

- [ ] **Step 2 : Modifier `runBin()` pour utiliser `priv()`**

Remplacer :

```typescript
private async runBin(bin: string, args: string[], largeOutput = false): Promise<F2bClientResult> {
    try {
        const { stdout } = await execFileAsync(bin, args, { timeout: EXEC_TIMEOUT, maxBuffer: largeOutput ? MAX_BUF_LARGE : MAX_BUF_SMALL });
        return { ok: true, output: stdout.trim() };
    } catch (err: unknown) {
        const msg = err instanceof Error ? err.message : String(err);
        const missing = msg.includes('ENOENT') || msg.includes('not found');
        return {
            ok: false, output: '',
            error: missing
                ? `${bin} non disponible dans le container.\nAjoutez dans docker-compose.yml :\n  cap_add:\n    - NET_ADMIN\nEt dans Dockerfile :\n  RUN apk add --no-cache iptables ipset nftables`
                : msg,
        };
    }
}
```

Par :

```typescript
private async runBin(bin: string, args: string[], largeOutput = false): Promise<F2bClientResult> {
    const [c, a] = priv(bin, args);
    try {
        const { stdout } = await execFileAsync(c, a, { timeout: EXEC_TIMEOUT, maxBuffer: largeOutput ? MAX_BUF_LARGE : MAX_BUF_SMALL });
        return { ok: true, output: stdout.trim() };
    } catch (err: unknown) {
        const msg = err instanceof Error ? err.message : String(err);
        const missing = msg.includes('ENOENT') || msg.includes('not found');
        return {
            ok: false, output: '',
            error: missing
                ? `${bin} non disponible.\nVérifiez que network_mode: host et cap_add: NET_ADMIN sont actifs dans docker-compose.yml.`
                : msg,
        };
    }
}
```
